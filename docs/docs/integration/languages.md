# Language Integration

Different programming languages have different best practices for version information. This guide covers recommended approaches for each.

## Choosing an Approach

| Language Type | Recommended Approach | Why |
|---------------|---------------------|-----|
| **Interpreted** (Python, Ruby, JS) | `versionator emit` | No compilation; generates source file at build time |
| **Compiled** (Go, Rust, C/C++) | Build-time injection | Avoids generated files in source control |

## Interpreted Languages

### Python

Generate a version file at build time:

```bash
versionator emit python --output mypackage/_version.py
```

Generated file:

```python
# Auto-generated by versionator
__version__ = "1.2.3"
__version_info__ = (1, 2, 3)
```

Use in your package:

```python
# mypackage/__init__.py
from ._version import __version__

# setup.py
from mypackage import __version__
setup(version=__version__)
```

Add to `.gitignore`:

```
mypackage/_version.py
```

### JavaScript/TypeScript

Generate version file:

```bash
# JavaScript
versionator emit js --output src/version.js

# TypeScript
versionator emit ts --output src/version.ts
```

Generated TypeScript:

```typescript
export const VERSION = "1.2.3";
export const VERSION_INFO = {
  major: 1,
  minor: 2,
  patch: 3,
};
```

Use in code:

```typescript
import { VERSION } from './version';
console.log(`App version: ${VERSION}`);
```

### Ruby

```bash
versionator emit ruby --output lib/myapp/version.rb
```

Generated file:

```ruby
module MyApp
  VERSION = "1.2.3"
end
```

### PHP

```bash
versionator emit php --output src/Version.php
```

Generated file:

```php
<?php
namespace MyApp;

class Version {
    public const VERSION = "1.2.3";
    public const MAJOR = 1;
    public const MINOR = 2;
    public const PATCH = 3;
}
```

## Compiled Languages

### Go

Inject version at compile time using `-ldflags`:

```go
// main.go
package main

var VERSION = "dev"

func main() {
    println("Version:", VERSION)
}
```

Build with version:

```bash
VERSION=$(versionator version)
go build -ldflags "-X main.VERSION=$VERSION" -o app
```

For packages:

```go
// internal/version/version.go
package version

var Version = "dev"
```

```bash
go build -ldflags "-X myapp/internal/version.Version=$VERSION" -o app
```

### Rust

Use environment variables at compile time:

```rust
// build.rs
use std::process::Command;

fn main() {
    let output = Command::new("versionator")
        .arg("version")
        .output()
        .expect("Failed to get version");

    let version = String::from_utf8(output.stdout)
        .expect("Invalid UTF-8")
        .trim()
        .to_string();

    println!("cargo:rustc-env=VERSION={}", version);
}
```

```rust
// main.rs
const VERSION: &str = env!("VERSION");

fn main() {
    println!("Version: {}", VERSION);
}
```

Or generate a file:

```bash
versionator emit rust --output src/version.rs
```

### C/C++

Pass as compiler define:

```bash
VERSION=$(versionator version)
gcc -DVERSION="\"$VERSION\"" -o app main.c
```

Or generate header:

```bash
versionator emit c-header --output include/version.h
```

Generated header:

```c
#ifndef VERSION_H
#define VERSION_H

#define VERSION "1.2.3"
#define VERSION_MAJOR 1
#define VERSION_MINOR 2
#define VERSION_PATCH 3

#endif
```

### Java

Generate at build time:

```bash
versionator emit java --output src/main/java/com/example/Version.java
```

Or use Maven/Gradle filtering with:

```bash
versionator version > version.txt
```

### Kotlin

```bash
versionator emit kotlin --output src/main/kotlin/Version.kt
```

### C#

```bash
versionator emit csharp --output src/Version.cs
```

### Swift

```bash
versionator emit swift --output Sources/Version.swift
```

## JSON/YAML

For configuration or API responses:

```bash
# JSON
versionator emit json --output version.json

# YAML
versionator emit yaml --output version.yml
```

JSON output:

```json
{
  "version": "1.2.3",
  "major": 1,
  "minor": 2,
  "patch": 3
}
```

## Custom Templates

Dump and customize built-in templates:

```bash
# Dump Python template
versionator emit dump python > custom_python.tmpl

# Edit custom_python.tmpl...

# Use custom template
versionator emit --template-file custom_python.tmpl --output _version.py
```

## Build Integration

### Makefile

```makefile
VERSION := $(shell versionator version)

# Interpreted: generate file
generate:
	versionator emit python --output src/_version.py

# Compiled: inject at build
build:
	go build -ldflags "-X main.VERSION=$(VERSION)" -o app
```

### Just

```just
version := `versionator version`

generate:
    versionator emit python --output src/_version.py

build:
    go build -ldflags "-X main.VERSION={{version}}" -o app
```

## Best Practices

1. **Add generated files to .gitignore**: Don't commit version files
2. **Generate at build time**: Run emit in build scripts, not manually
3. **Use appropriate approach**: Inject for compiled, generate for interpreted
4. **Include in CI**: Ensure version files are generated in CI/CD
