# Python Integration

This guide shows how to integrate versionator with a Python project.

## Project Structure

```
myproject/
├── VERSION
├── .versionator.yaml
├── Makefile
├── setup.py
├── pyproject.toml
└── mypackage/
    ├── __init__.py
    └── _version.py        # Generated
```

## Setup

### Initialize Version

```bash
cd myproject
versionator version
# Creates VERSION with 0.0.0
```

### Configure Versionator

```yaml
# .versionator.yaml
prefix: ""  # Python doesn't typically use 'v' prefix
```

## Generate Version File

Create the version file:

```bash
versionator emit python --output mypackage/_version.py
```

Generated content:

```python
# Auto-generated by versionator
__version__ = "1.2.3"
__version_info__ = (1, 2, 3)
```

## Package Integration

### `__init__.py`

```python
# mypackage/__init__.py
from ._version import __version__, __version_info__

__all__ = ['__version__', '__version_info__']
```

### `setup.py`

```python
# setup.py
from mypackage import __version__

setup(
    name='mypackage',
    version=__version__,
    # ...
)
```

### `pyproject.toml` (setuptools)

```toml
[project]
dynamic = ["version"]

[tool.setuptools.dynamic]
version = {attr = "mypackage.__version__"}
```

## Build Automation

### Makefile

```makefile
.PHONY: version build test publish

VERSION := $(shell versionator version)

version:
	versionator emit python --output mypackage/_version.py

build: version
	python -m build

test: version
	pytest

publish: build
	twine upload dist/*

bump-patch:
	versionator patch increment
	$(MAKE) version

bump-minor:
	versionator minor increment
	$(MAKE) version

bump-major:
	versionator major increment
	$(MAKE) version

release: bump-patch
	git add VERSION mypackage/_version.py
	git commit -m "Release $(VERSION)"
	versionator tag
	git push --tags
	$(MAKE) publish
```

### Just

```just
version := `versionator version`

# Generate version file
emit-version:
    versionator emit python --output mypackage/_version.py

# Build package
build: emit-version
    python -m build

# Run tests
test: emit-version
    pytest

# Bump and release
release bump="patch":
    versionator {{bump}} increment
    just emit-version
    git add VERSION mypackage/_version.py
    git commit -m "Release $(versionator version)"
    versionator tag
    git push --tags
    just build
    twine upload dist/*
```

## CI/CD Integration

### GitHub Actions

```yaml
name: Python Package

on:
  push:
    tags:
      - '*.*.*'

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Install versionator
        run: go install github.com/benjaminabbitt/versionator@latest

      - name: Generate version file
        run: versionator emit python --output mypackage/_version.py

      - name: Install build dependencies
        run: pip install build twine

      - name: Build package
        run: python -m build

      - name: Publish to PyPI
        env:
          TWINE_USERNAME: __token__
          TWINE_PASSWORD: ${{ secrets.PYPI_TOKEN }}
        run: twine upload dist/*
```

## Custom Template

For more control, customize the template:

```bash
# Dump default template
versionator emit dump python > python_version.tmpl
```

Edit `python_version.tmpl`:

```python
# {{AppName}} version file
# Generated by versionator - do not edit manually

__version__ = "{{MajorMinorPatch}}"
__version_info__ = ({{Major}}, {{Minor}}, {{Patch}})
__author__ = "{{CommitAuthor}}"
__build_date__ = "{{BuildDateUTC}}"
```

Use custom template:

```bash
versionator emit --template-file python_version.tmpl \
    --output mypackage/_version.py \
    --set AppName="MyApp"
```

## Runtime Access

In your code:

```python
from mypackage import __version__

def main():
    print(f"MyApp version {__version__}")

if __name__ == "__main__":
    main()
```
