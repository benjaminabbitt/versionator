Version Grammar

Railroad diagrams and EBNF grammar for version string parsing.

Supported Formats

SemVer 2.0.0
1.2.3 1.0.0-alpha 1.0.0-alpha.1 1.0.0+build.123 1.0.0-beta+exp.sha.5114f85
With Prefix
v1.2.3 V1.0.0-rc.1
Partial Versions
1 1.2
Go Pseudo-versions
v0.0.0-20191109021931-daa7c04131f5
Assembly Versions
1.0.0.0 2.1.3.456

EBNF Grammar

VersionFile = Version .
Version = ? VersionCore PreRelease? BuildMetadata? .
VersionCore =  (  (  ( )?)?)? .
PreRelease =  Identifier ( Identifier)* .
Identifier =  |  |  .
BuildMetadata =  Identifier ( Identifier)* .

Lexer Tokens

Token Pattern Description
Prefix[vV]Version prefix (v or V)
Number[0-9]+Numeric component
Dot.Component separator
Dash-Pre-release prefix
Plus+Build metadata prefix
Ident[a-zA-Z0-9-]+Identifier (alphanumeric with dashes)

Precedence Rules

  1. Compare major, minor, patch numerically (left to right)
  2. Pre-release has lower precedence than normal: 1.0.0-alpha < 1.0.0
  3. Pre-release identifiers compared left to right:
    • Numeric identifiers compared as integers
    • Alphanumeric compared as ASCII sort
    • Numeric < Alphanumeric
    • Shorter set < longer set
  4. Build metadata is ignored in precedence comparisons

Ascending order example:

1.0.0-alpha 1.0.0-alpha.1 1.0.0-alpha.beta 1.0.0-beta 1.0.0-beta.2 1.0.0-beta.11 1.0.0-rc.1 1.0.0

Invalid Examples

1.2.3.4.5 1.02.3 1.2.3- 1.2.3+ 1.2.3-alpha..1 v-1.2.3
